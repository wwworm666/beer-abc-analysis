# Comparison Module Redesign — Modern Fintech UX

> Превращаем базовое сравнение периодов в продвинутый финтех-инструмент

---

## Проблемы текущей версии

### UX Issues
- ❌ Нет умных дефолтов — пользователь должен вручную выбирать оба периода
- ❌ Кнопки "vs предыдущая неделя" работают только если уже выбран период
- ❌ Селекторы выглядят устаревшими (plain HTML select)
- ❌ Нет визуальной иерархии — всё одинаково важно
- ❌ Загрузка показывается спиннером (не современно)

### Визуальные проблемы
- ❌ Таблица выглядит как Excel 2003
- ❌ График базовый, без кастомизации
- ❌ Инсайты — просто текст в рамках
- ❌ Нет микроанимаций и transitions
- ❌ Цвета плоские, нет градиентов

### Функциональные ограничения
- ❌ Можно сравнить только 2 периода
- ❌ Нет drill-down в детали
- ❌ Нет экспорта сравнения
- ❌ Нет трендов и прогнозов
- ❌ Нет AI-инсайтов (хотя в других модулях есть)
- ❌ Нельзя сравнить несколько баров одновременно

## Концепция нового дизайна

### 1. Hero Section — Умный выбор периодов

**Вместо двух селекторов:**

```
┌─────────────────────────────────────────────────────────┐
│  Сравнение периодов                                      │
│                                                          │
│  [Текущая неделя ▼]  vs  [Предыдущая неделя ▼]         │
│                                                          │
│  Быстрые сравнения:                                      │
│  ○ Неделя к неделе   ● Месяц к месяцу   ○ Год к году   │
│                                                          │
│  📅 Или выберите произвольный период                     │
└─────────────────────────────────────────────────────────┘
```

**Новый подход:**

```
┌─────────────────────────────────────────────────────────────────┐
│  📊 Сравнение периодов                                          │
│                                                                 │
│  ┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐│
│  │ Week-over-Week  │  │ Month-over-Month│  │ Year-over-Year  ││
│  │   (активно)     │  │                 │  │                 ││
│  └─────────────────┘  └─────────────────┘  └─────────────────┘│
│                                                                 │
│  Период 1: [10-16 марта 2026 ▼]                               │
│  Период 2: [3-9 марта 2026 ▼]    ← автоматически выбран       │
│                                                                 │
│  🎯 Сравнить по: [Все бары ▼] [Все категории ▼]               │
└─────────────────────────────────────────────────────────────────┘
```

**Фичи:**
- Автоматический выбор периодов при загрузке (текущая vs предыдущая неделя)
- Табы для быстрого переключения режима сравнения
- Фильтры по барам и категориям
- Flatpickr с range mode для кастомных периодов

### 4. Waterfall Chart — Анализ изменений

**Вместо простого bar chart:**

```
┌─────────────────────────────────────────────────────────────┐
│  Анализ изменения выручки                                   │
│                                                             │
│         ┌─────┐                                             │
│  1.2M   │     │  ┌───┐                                      │
│         │     │  │+50│  ┌───┐                               │
│  1.1M   │     │  └───┘  │-20│  ┌───┐                        │
│         │     │         └───┘  │+30│  ┌─────┐              │
│  1.0M   │     │                └───┘  │     │              │
│         └─────┘                       └─────┘              │
│         Период 1  Розлив  Фасовка  Кухня  Период 2         │
│                   ↑ +50K  ↓ -20K   ↑ +30K                  │
└─────────────────────────────────────────────────────────────┘
```

**Фичи:**
- Показывает вклад каждой категории в изменение
- Цветовая кодировка (зелёный рост, красный падение)
- Интерактивный (click на сегмент → детали)

## Технический стек

### Frontend

**Новые библиотеки:**
- `chart.js` → уже есть, добавить waterfall plugin
- `sparklines.js` или `peity.js` — для мини-графиков в карточках
- `gsap` или `framer-motion` — для анимаций
- `html2canvas` — для screenshot export

**CSS:**
- Использовать существующий design system (IBM Plex Mono, accent #D97757)
- Добавить CSS переменные для comparison-специфичных цветов
- Glassmorphism эффекты для карточек
- Smooth transitions (0.3s ease)

### Backend

**Новые endpoints:**

```python
# Получить данные для multi-period comparison
POST /api/comparison/multi-period
{
  "venue_key": "kremenchugskaya",
  "periods": [
    {"start": "2026-03-10", "end": "2026-03-16"},
    {"start": "2026-03-03", "end": "2026-03-09"},
    {"start": "2026-02-24", "end": "2026-03-02"}
  ]
}

# Получить AI insights для сравнения
POST /api/comparison/insights
{
  "venue_key": "kremenchugskaya",
  "period1": {"start": "2026-03-10", "end": "2026-03-16"},
  "period2": {"start": "2026-03-03", "end": "2026-03-09"}
}

# Экспорт сравнения
POST /api/comparison/export
{
  "format": "pdf",  # pdf | excel | png
  "data": { ... }
}
```

**AI Insights генерация:**
- Использовать существующий паттерн из dashboard (AI-анализ уже есть)
- Добавить anomaly detection (2σ от среднего)
- Категоризация изменений (operational, seasonal, anomaly)
- Recommendations engine

## Референсы для вдохновения

### Fintech Products
- **Stripe Dashboard** — чистый дизайн, отличные таблицы

- **Wise Business** — comparison sliders

### Design Patterns
- **Metric Cards**: большое число + контекст + sparkline
- **Comparison Slider**: stacked bars с процентами
- **Waterfall Chart**: для анализа изменений
- **AI Panel**: категоризация + recommendations
- **Progressive Disclosure**: summary → details on demand
