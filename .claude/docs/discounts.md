# Анализ акций: Скидки и RFM-сегментация

## Что это

Единая страница с двумя режимами работы:
- **Скидки** — анализ эффективности скидок/акций
- **RFM-сегментация** — сегментация гостей по давности, частоте и среднему чеку

Переключатель режимов находится в верхней части страницы.

---

## Режим "Скидки"

### Назначение
Оценить эффективность конкретной скидки или акции. Показать, кто использовал скидку, на какую сумму, и дать краткие инсайты о профиле гостей.

### Интерфейс

**Контролы:**
- Бар — выбор конкретной точки или "Все бары"
- Период — диапазон дат
- Скидка — название акции/скидки из dropdown
- Кнопка "Анализировать"

**Отображение:**
1. **5 базовых метрик:**
   - Выручка без скидки
   - Выручка со скидкой
   - Отдали скидками
   - Глубина скидки %
   - Средний чек

2. **Таблица гостей** (использовавших скидку):
   - №, Гость, Карта, Визитов, Выручка, Скидка
   - Клик по строке → детали гостя

3. **Профиль гостей скидки** (3 инсайта):
   - Средний recency (дней с последнего визита)
   - % ушедших гостей (60+ дней)
   - Средняя частота визитов (раз/неделю)

### Формулы

| Метрика | Формула |
|---------|---------|
| Выручка без скидки | `DishDiscountSumInt + DiscountSum` |
| Выручка со скидкой | `DishDiscountSumInt` |
| Глубина скидки | `DiscountSum / (DishDiscountSumInt + DiscountSum) × 100%` |
| Средний чек | `DishDiscountSumInt / кол-во чеков` |
| Средний recency | `Среднее(recency_days)` по всем гостям |
| % ушедших | `Кол-во гостей с recency > 60 / Всего гостей × 100%` |
| Средняя частота | `Среднее(frequency_per_week)` по всем гостям |

## Режим "RFM-сегментация"

### Назначение
Сегментировать всю базу гостей для маркетинговых кампаний:
- Выявить ушедших гостей для winback-рассылок
- Найти чемпионов для эксклюзивных предложений
- Отследить паттерны посещения

### Интерфейс

**Контролы:**
- Период — диапазон дат (по умолчанию: с начала года по сегодня)
- Бар — выбор конкретной точки или "Все бары"
- Кнопка "Обновить"

**Отображение:**

1. **6 карточек сегментов:**
   - Чемпионы, Лояльные, В риске, Ушедшие, Новички, Потенциальные
   - Количество гостей и % от базы

2. **7 метрик базы гостей:**
   - Всего гостей
   - Активных (0-30 дней)
   - Ушедших (60+ дней)
   - Средний recency
   - Средняя частота
   - Выручка
   - Отдали скидками

3. **Таблица всех гостей:**
   - Статус (сегмент с цветовой индикацией)
   - Гость, Карта, Посл. визит, Recency, Частота, Визитов, Ср. чек, Выручка
   - Фильтр по сегменту (dropdown)
   - Кнопка "Экспорт CSV"

4. **Визуализации (3 вкладки):**
   - **Гистограмма** — распределение по recency (5 бакетов)
   - **Scatter plot** — Frequency (X) vs Recency (Y), размер точки = средний чек
   - **Сегменты** — детальные карточки с описанием и выручкой

### RFM-сегменты

**Recency (давность последнего визита):**
| Код | Дней | Название |
|-----|------|----------|
| R5 | 0-7 | Активные |
| R4 | 8-14 | Recent |
| R3 | 15-30 | Средний риск |
| R2 | 31-60 | Высокий риск |
| R1 | 60+ | Ушедшие |

**Frequency (частота визитов в неделю):**
| Код | Визитов/нед | Название |
|-----|-------------|----------|
| F5 | 3+ | VIP |
| F4 | 2-3 | Лояльные |
| F3 | 0.8-2 | Регулярные |
| F2 | 0.3-0.8 | Редкие |
| F1 | <0.3 | Очень редкие |

**Комбинированные сегменты:**
| Сегмент | Критерии | Описание |
|---------|----------|----------|
| CHAMPIONS | R5 + F4-5 | Ходят часто и недавно |
| LOYAL | R4-5 + F3-4 | Стабильные гости |
| AT_RISK | R2-3 + F3-5 | Раньше ходили часто, сейчас реже |
| CHURNED | R1 + F3-5 | Не ходят 60+ дней (были активны) |
| NEW | R5 + visits=1 | Первый визит <7 дней назад |
| POTENTIAL | R4-5 + F1-2 | Редкие, но свежие |

### Формулы

| Метрика | Формула |
|---------|---------|
| `recency_days` | `(period_end - last_visit).days` |
| `frequency_per_week` | `total_visits / period_days × 7` |
| `avg_check` | `sum_with_discount / total_visits` |
| Активные гости | `recency_days <= 30` |
| Ушедшие гости | `recency_days > 60` |

## Как работает

### Архитектура

**OLAP-запросы:**

Для скидок (`get_discount_report`):
```python
groupByRowFields: [
    "Store.Name",
    "Delivery.CustomerCardNumber",
    "Delivery.CustomerName",
    "OrderNum",
    "DishName",
    "ItemSaleEventDiscountType",
    "OpenDate.Typed"
]
```

Для RFM (`get_rfm_report`):
```python
groupByRowFields: [
    "Store.Name",
    "Delivery.CustomerCardNumber",
    "Delivery.CustomerName",
    "OrderNum",
    "OpenDate.Typed"
]
# Без ItemSaleEventDiscountType и DishName — все транзакции
```

### Бэкенд: расчёт метрик

Для каждого гостя рассчитываются:
- `visit_dates` — список уникальных дат визитов
- `last_visit` — дата последнего визита
- `first_visit` — дата первого визита
- `recency_days` — дней с последнего визита
- `frequency_per_week` — визитов в неделю
- `avg_check` — средний чек

### Фронтенд: переключение режимов

```javascript
function switchMode(mode) {
    // Обновляет активную кнопку
    // Показывает/скрывает контролы

    // Обновляет URL (?mode=rfm)
}
```

## Сценарии использования

### Сценарий 1: Анализ эффективности акции
1. Режим: **Скидки**
2. Выбрать скидку: "Чёрная пятница -20%"
3. Выбрать период: неделя акции
4. Нажать "Анализировать"
5. Посмотреть: выручка, глубина скидки, профиль гостей

### Сценарий 2: Поиск ушедших гостей для winback-кампании
1. Режим: **RFM-сегментация**
2. Выбрать период: последние 6 месяцев
3. Фильтр: Сегмент = Ушедшие
4. Нажать "Экспорт CSV"
5. Использовать список для рассылки

### Сценарий 3: Оценка лояльности гостей бара
1. Режим: **RFM-сегментация**
2. Выбрать бар: конкретная точка
3. Посмотреть карточки сегментов:
   - Сколько чемпионов (ходят часто и недавно)
   - Сколько ушедших (требуют внимания)
4. Вкладка "Scatter plot": визуальная оценка распределения
