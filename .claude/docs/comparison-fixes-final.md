# Comparison Module — Final Fixes

## Проблемы и решения

### 1. Шкалы (progress bars) не к чему не привязаны ✅

**Было:** Progress bar показывал произвольное значение без смысла

**Стало:** Progress bar показывает отношение Period 1 к Period 2 (как в dashboard: факт/план)

```javascript
// Процент для progress bar: val1/val2 * 100
const progressPercent = val2 !== 0 ? (val1 / val2) * 100 : 0;
const progressWidth = Math.min(progressPercent, 100);
```

**Пример:**
- Period 2: 100,000 ₽
- Period 1: 120,000 ₽
- Progress bar: 120% → отображается как 100% (заполнен полностью)

### 2. Не понятно какой показатель был, стал и процент ✅

**Было:** Непонятная структура footer

**Стало:** Чёткая структура с тремя элементами:

```html
<div class="metric-footer">
    <span class="metric-percentage">+20.0%</span>      <!-- процент изменения -->
    <span class="metric-deviation">+20 000 ₽</span>   <!-- абсолютная разница -->
    <span class="metric-plan">было 100 000 ₽</span>   <!-- предыдущее значение -->
</div>
```

**Карточка показывает:**
- **Большое значение:** текущий период (Period 1)
- **Progress bar:** отношение текущего к предыдущему
- **Footer:**
  - Процент изменения: `+20.0%` или `-15.5%`
  - Абсолютная разница: `+20 000 ₽` или `-5 000 ₽`
  - Предыдущее значение: `было 100 000 ₽`

### 3. Проценты в наценке считаются неправильно ✅

**Проблема 1:** Отображение `19300.0%` вместо `193.0%`

API возвращает значения уже в процентах (193 = 193%), но formatter умножал на 100:
```javascript
// БЫЛО (неправильно)
formatter: (v) => `${(v * 100).toFixed(1)}%`  // 193 * 100 = 19300%
```

**Проблема 2:** Разница показывалась неправильно

Думали что значения в десятичных (0.25), умножали на 100, но они уже в процентах (25):
```javascript
// БЫЛО (неправильно)
formattedDiff = `${(diff * 100).toFixed(1)} п.п.`;  // 48 * 100 = 4800 п.п.
```

**Решение:**

1. Используем `formatPercent` из utils.js (не умножает на 100):
```javascript
// СТАЛО (правильно)
{ key: 'markupDraft', altKey: 'draft_markup', label: 'Наценка розлив',
  formatter: formatPercent }
```

2. Не умножаем разницу на 100:
```javascript
// СТАЛО (правильно)
formattedDiff = `${Math.abs(diff).toFixed(1)} п.п.`;  // 48 п.п.
```

**Пример:**
- Period 2: 193% (приходит как 193)
- Period 1: 241% (приходит как 241)
- Разница: 48 → **+48.0 п.п.** ✓ (не 4800 п.п.)

## Логика расчётов

### Процент изменения (для footer)
```javascript
const changePercent = val2 !== 0 ? ((diff / val2) * 100) : 0;
```

Показывает: "на сколько процентов изменилось значение"

**Пример:**
- Было: 100,000 ₽
- Стало: 120,000 ₽
- Изменение: +20.0%

### Процент для progress bar
```javascript
const progressPercent = val2 !== 0 ? (val1 / val2) * 100 : 0;
```

Показывает: "какую долю от предыдущего периода составляет текущий"

**Пример:**
- Было: 100,000 ₽
- Стало: 120,000 ₽
- Progress: 120% (отображается как 100%, т.к. max)

### Статус (цвет индикатора)
```javascript
let statusClass = 'neutral';
if (Math.abs(changePercent) > 1) {
    statusClass = diff > 0 ? 'success' : 'danger';
}
```

- **success** (зелёный): рост > 1%
- **danger** (красный): падение > 1%
- **neutral** (серый): изменение ≤ 1%
