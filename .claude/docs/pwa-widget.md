# PWA Виджет Выручки

## Что это

PWA (Progressive Web App) виджет для отображения метрик выручки по барам. Позволяет пользователям быстро проверять выполнение плана по 5 барам на мобильном устройстве.

**Важно:** Это PWA-приложение (ярлык на главный экран), а не нативный виджет Android/iOS. Для настоящего виджета с данными на главном экране требуется отдельное нативное приложение (Kotlin для Android, Swift для iOS).

### API Endpoint

**GET `/api/widget/revenue`**

Возвращает упрощённые данные для 5 баров (только процент выполнения):

```json
[
  {
    "bar": "",
    "name": "Общая",
    "completion": 83.1
  },
  {
    "bar": "bolshoy",
    "name": "Большой пр. В.О",
    "completion": 80.8
  },
  {
    "bar": "ligovskiy",
    "name": "Лиговский",
    "completion": 92.1
  }
]
```

**Логика расчёта:**
1. Период: с 1-го числа текущего месяца по сегодня
2. Для каждого бара вызывается тот же OLAP-запрос, что и в дашборде
3. План берётся из `data/plansdashboard.json`
4. Вычисляется `% выполнения = факт / план × 100`
5. Результат кэшируется на 5 минут

### Детальный экран

При тапе на карточку бара вызывается существующий endpoint:

**POST `/api/revenue-metrics`**

```json
{
  "bar": "bolshoy",
  "date_from": "2026-03-01",
  "date_to": "2026-03-26"
}
```

Ответ:
```json
{
  "current": 1234567.89,
  "plan": 1500000.00,
  "expected": 1456789.00,
  "average": 56789.12,
  "completion_percent": 82.3
}
```

### Service Worker

Стратегия кэширования:
- **Статика (CSS, JS, fonts):** Cache-first
- **API запросы:** Network-first с fallback на кэш
- **HTML страницы:** Network-first

Это позволяет виджету работать офлайн (показывает кэшированные данные).

### Установка на устройство

**Android (Chrome):**
1. Открыть `https://beer-abc-analysis.onrender.com/dashboard/widget`
2. Меню (⋮) → "Добавить на главный экран" или "Установить приложение"

**iOS (Safari):**

## UI/UX Решения

### Цветовая индикация
- **≥100%** — зелёный (success) ✅
- **≥80%** — жёлтый (warning) ⚠️
- **<80%** — красный (danger) ❌

### Адаптивность
- **Десктоп:** 5 карточек в ряд
- **Планшет (≤1400px):** 3 карточки в ряд
- **Мобильный (≤900px):** 2 карточки в ряд
- **Маленький (≤640px):** 1 карточка

### Анимации
- Hover: `transform: scale(1.02)` + elevation shadow
- Progress bar: `transition: width 0.3s cubic-bezier(0.4, 0, 0.2, 1)`
- Install prompt: `animation: slideUp 0.3s`

### Доступность
- Touch targets: минимум 44×44px
- Focus visible: `outline: 2px solid var(--accent)`
- Reduced motion: отключение анимаций при `prefers-reduced-motion`
- Color contrast: 4.5:1 минимум

## Почему не нативный виджет?

| Критерий | PWA | Нативный Android | Нативный iOS |
|----------|-----|------------------|--------------|
| Время разработки | 1 день | 2-4 недели | 2-4 недели |
| Стоимость | Бесплатно | Бесплатно | $99/год (App Store) |
| Кодбаза | Одна | Отдельная | Отдельная |
| Обновления | Мгновенно | Через APK/Store | Через App Store |
| Данные на экране | ❌ (ярлык) | ✅ | ✅ |

**Решение:** PWA выбрано как fastest path to value. Для настоящего виджета с данными на главном экране потребуется отдельное нативное приложение (Kotlin/Swift).
