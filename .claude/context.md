# Context для Claude

## Быстрый старт

Это система аналитики для 4 пивных баров в Санкт-Петербурге:
- Большой пр. В.О (24 крана)
- Лиговский (12 кранов)
- Кременчугская (12 кранов)
- Варшавская (12 кранов)

## Ключевые концепции

### iiko
POS-система (касса). Все продажи, остатки, сотрудники — там. API работает через XML/JSON.

### OLAP
Отчёты из iiko. Медленные (5-15 сек), но детальные. Кэшируем где можем.

### Кассовые смены (Cash Shifts)
`/v2/cashshifts/list` — кто работал, где, когда. Один бармен = одна смена.
`responsibleUserId` = бармен, `pointOfSaleId` = бар.

### План/Факт сотрудников
План = Σ(план_ТТ[дата]) за дни когда сотрудник работал.
Данные о сменах из Cash Shifts, планы из `data/daily_plans.json`.

### Карты лояльности
Новые регистрации считаем по `Delivery.CustomerPhone` (уникальные телефоны) с фильтром по `Delivery.CustomerCreatedDateTyped` (дата создания клиента в периоде).
**НЕ** использовать `CardNumber` — это банковские карты, а не лояльность.

### Нормализация имён сотрудников
OLAP: "Васильев Никита", iiko employees: "Никита Васильев". Сравниваем по множеству слов (порядок неважен).

## Маппинги

```python
# Бары: ключ → iiko название
'bolshoy' → 'Большой пр. В.О'
'ligovskiy' → 'Лиговский'
'kremenchugskaya' → 'Кременчугская'  # она же "Пивная культура"
'varshavskaya' → 'Варшавская'

# Для кранов: bar_id
'bar1' = Большой
'bar2' = Лиговский
'bar3' = Кременчугская
'bar4' = Варшавская
```
