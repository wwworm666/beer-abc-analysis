## Киоск самообслуживания

В этой статье мы рассмотрим общую концепцию построения продукта. При разработке используйте API iikoFront.

* [Документация по API iikoFront](https://iiko.github.io/front.api.doc/)
* [Примеры реализации плагина на API](https://github.com/iiko/front.api.sdk/tree/master/sample)

Задать вопросы по разработке можно [здесь](https://github.com/iiko/front.api.sdk/issues).

### Что потребуется для разработки

1. Стационарный компьютер или ноутбук.

Для стенда рекомендуем использовать стационарный компьютер или ноутбук, а не виртуальную машину. Также может потребоваться дополнительное оборудование: сканер, принтер, банковский терминал.

Требования для стационарного терминала официанта/бармена/кассира:

* Процессор: intel Core i3, частота не менее 1.8GHz;
* Оперативная память: рекомендуется 4GB RAM;
* Свободное место на жестком диске: 80GB;
* Сеть: 100Mbit Ethernet;
* Разрешение дисплея: 1024х768;
* Операционная система Windows.

1. Демо-стенд c установленным ПО iiko.

Для настройки обратитесь к любому партнеру iiko в вашем регионе: лицензию вам предоставят на месяц бесплатно. Если месяца вам не хватит, чтобы попробовать продукт, лицензию могут продлить по запросу.

### 
Основные части продукта

Ваше будущее ПО будет представлять из себя интеграционное решение, в составе которого будут как стандартные компоненты iiko (iikoFront, iikoOffice), так и написанные на API модули и компоненты. Рассмотрим, из каких модулей будет состоять программно-аппаратный комплекс в ресторане:

* Сервер iikoRMS — app-сервер + база данных.
* iikoBackOffice — клиентское приложение .Net Winforms для сотрудников ресторана. С его помощью можно настроить ресторан, вводить номенклатуры, накладные, сотрудников, настраивать отчеты, контролировать продажи т.д.
* Касса iikoFront — клиентское приложение .Net WPF для кассиров, официантов. С его помощью создаются заказы внутри ресторана с последующей оплатой и печатью фискальных/нефискальных чеков.

![](/resources/Storage/api-documentations/scheme-1.PNG)
Основные модули в ресторане

* Киоск самообслуживания — компьютер в упрощенном виде. На этот компьютер устанавливается iikoFront в режиме агента, чтобы iikoFront автоматически запускался после загрузки ОС и работал в фоновом режиме. Киоск будет обладать всеми возможностями кассы. Важно, чтобы iikoFront был настроен именно как касса.

Для работы киоска нужно разработать плагин на C#, с помощью которого заказы из него будут передаваться в iikoFront. Внутри плагина должен web-сервис, который будет взаимодействовать с основным ПО киоска.

![](/resources/Storage/api-documentations/scheme-2.PNG)

Киоск самообслуживания

В плагине должны быть реализованы следующие EndPoint:

* Получение номенклатуры c ценой;
* Получить список столов;
* Создать заказ;
* Добавить тип оплаты;
* Оплатить;

Все эти методы API вы можете найти в [примере](https://github.com/iiko/front.api.sdk/tree/master/sample).

### Вопросы и ответы

С какой лицензией продавать продукт?

После реализации вашего ПО для киоска самообслуживания в плагине пропишите modul ID лицензии открытого API — 21016318. Modul ID содержится в лицензии API Payment Connector.
