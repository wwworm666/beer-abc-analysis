##  С чего начать 

### 1. Подготовьте меню 
 
Требования к меню:

* Во внешнее меню можно выгружать: товары, блюда, модификаторы и услуги.
* У каждого элемента должны быть: тип места приготовления и место продаж.

Чтобы подготовить меню для iikoTransport:

1. Выберите пункт **Обмен данными → Выгрузка меню.**
2. Сформируйте внешнее меню. Его структура может на совпадать со структурой номенклатуры. Модификаторы, которые продаются вместе с блюдом, не нужно добавлять в меню отдельно.
3. К каждому элементу можно прикрепить изображение. Оно не связано с изображением в номенклатурной карточке блюда. Размер изображения не должен превышать 10Мб.
4. Нажмите **Сохранить и закрыть.** Кнопку **"выгрузить"** нажимать не требуется, она используется только для выгрузки меню в кард.

Ограничения на размер внешнего меню бек офиса:

Меню выгружается в Cloud API в размере не превышающем 10 Мб (учитывается только текст в сформированном json без учета изображений). Значение стандартно для всех клиентов и не меняется. Если после изменения внешнего меню в бек офисе данные не уходят в cloud API (более подробно см [статью по диагностике состояния подключения к Cloud API](/smart/project-api-documentation/cloud-api-check)) - то нужно оценить масштаб внешнего меню и его вложенность и сократить меню: удалить старые неактуальные каталоги. Если у клиента очень большое меню на более чем 1000 позиций советуем создать 1 каталог и на группы внутри него задать теги для разных агрегаторов чтобы не было дублирования меню в каталогах либо перейти на внешнее меню веба для Cloud клиентов.

### 2. Авторизуйтесь в API iikoTransport 

Для авторизации в API iikoTransport используйте [метод получения авторизационного токена](https://api-ru.iiko.services/#tag/Authorization/paths/~1api~11~1access_token/post).

Передавайте полученный токен в поле access\_token в последующих методах.

### 3. Загрузите номенклатуру 

Вызовите [метод получения номенклатуры](https://api-ru.iiko.services/#tag/Menu/paths/~1api~11~1nomenclature/post). Укажите полученный ранее токен, требуемый таймаут и ID организации (или нескольких организаций), номенклатуру которых необходимо получать.

Тело запроса:


```json
{
"common": {
"accessToken": "{{publicapi_token}}",
"requestTimeout": "60"
},
"request": {
"organizationId": "{{organization_id}}"
}
}
```

```json
{
"common": {
"accessToken": "{{publicapi_token}}",
"requestTimeout": "60"
},
"request": {
"organizationId": "{{organization_id}}"
}
}
```


В ответ придёт номенклатура:


```json
{
"groups": [ ],
"productCategories": [],
"products": [],
"revision": 6635,
"uploadDate": "2018-06-27 15:28:51"
}
```

```json
{
"groups": [ ],
"productCategories": [],
"products": [],
"revision": 6635,
"uploadDate": "2018-06-27 15:28:51"
}
```


В поле **groups** содержатся:

* группы номенклатуры во внешнем меню.
* групповые модификаторы в выгружаемой номенклатуре.

Важные изменения по вводу ограничений для метода запроса номенклатуры из внешнего меню РМС: /api/1/nomenclature:
 Заметили, что в начале каждого часа большая часть интеграций выполняет запрос номенклатуры v1 по сетям. Происходит это одномоментно, что создает существенную нагрузку на базу транспорта.
В Cloud API ужесточили ограничения на запрос номенклатуры v1 - Транспорт больше не позволяет делать такие параллельные запросы.
Правильное поведение:
- **Запрашивать метод по ревизии**. Получить первый раз номенклатуру с нулевой ревизией и сохранить ревизию вернувшуюся в ответе. Следующий раз запрашивать уже по сохраненной ревизии и т.д. **ВАЖНО**: **с нулевой ревизией** можно запросить номенклатуру только с периодичностью**один запрос по одному РМС на 1 апи логин в час**. (**UPD**: временной слот для неревизионного запроса сменился 13.02 с 1 минуты на 1 час)
- В дальнейшем запросы получения номенклатуры **распределять по времени** - в одну минуту от одного апи логина запрашивать номенклатуру не более чем по 3м организациям с указанной ревизией. Если сеть больше - то через минуту новую порцию 3шт и тд. Важно: запрашивать номенклатуру не следует чаще чем раз в полчаса-час по одной точке. Данный справочник не так часто меняется.

Пример ответа с кодом 429 при достижении лимита:

```json
{   
  "correlationId": "***",    
  "errorDescription":   "Too many requests without revision within allowed period of time.",   
  "error": "TOO_MANY_NOMENCLATURE_REQUESTS_WITHOUT_REVISION"
}
```

```json
{   
  "correlationId": "***",    
  "errorDescription":   "Too many requests without revision within allowed period of time.",   
  "error": "TOO_MANY_NOMENCLATURE_REQUESTS_WITHOUT_REVISION"
}
```
