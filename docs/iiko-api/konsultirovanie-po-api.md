Чтобы получить помощь с разработкой Вашего проекта, можно приобрести пакет iikoIntegratorsKit. У нас есть два варианта:

## iikoIntegratorsKit **Consulting** 

В этом варианте мы оказываем только консультационное сопровождение проекта. Данный Kit подойдет тем, кто создает приложения доставки, сервисы по резервированию столов, заказу банкетов, оплате у стола и т.п.

Данный пакет включает в себя следующие услуги:
* разворачивание демо-стенда в облаке и выдачу демо-лицензии iiko;
* помощь с настройкой дополнительных модулей iiko;
* помощь с решением технических проблем API в приоритетном порядке (через чат в Телеграм);
* рассмотрение и обсуждение архитектуры интеграции, создание WorkFlow, рекомендации по использованию тех или иных методов API;
* помощь в понимании тех или иных методов API;
* предоставление Postman-коллекции (при необходимости);
* помощь в создании пользовательской документации.

## iikoIntegratorsKit **Coworking** 

В данном варианте на проект выделяется **разработчик**, который участвует в разборе исходного кода проекта. Данный Kit подойдет, если вы подключаете новое оборудование, разрабатываете платежные сервисы, интеграции с гостиничными системами или системами лояльности, а также системы оперативного мониторинга.

Данный пакет включает в себя следующие услуги:
* разворачивание демо-стенда в облаке, выдачу демо-лицензии iiko;
* помощь с настройкой дополнительных модулей iiko;
* помощь с решением технических проблем API в приоритетном порядке (через чат в Телеграм);
* рассмотрение и обсуждение архитектуры интеграции, создание WorkFlow, рекомендации по использованию тех или иных методов API;
* помощь в понимании тех или иных методов API и внутренней архитектуры кассы;
* разбор исходного кода проекта (плагина) выделенным разработчиком;
* помощь в создании пользовательской документации.

## Подключение iikoIntegratorsKit 
Есть два варианта подключения iikoIntegratorsKit (стоимость уточняется по актуальному прайс-листу в зависимости от количества месяцев поддержки).

### Обратиться на почту [api@iiko.ru](mailto:api@iiko.ru) 

Алгоритм подключения в данном случае будет такой:
* организация звонка с командой API для понимания сути проекта и обсуждения процесса в целом (обсуждение необходимости приобретения Kit или ее отсутствие);
* определение срока, на который будет приобретен kit, а также общего roadmap;
* подписание договора и выставление счета;
* организация ТГ канала, где решаются только быстрые вопросы;
* подключение к Pyrus для решения остальных вопросов;
* по окончании срока договора на kit, организация финального звонка и обсуждение результатов;
* отметка о завершении работы с клиентом.

### Обратиться к партнеру 
В этом случае алгоритм подключения выглядит так:* партнер заполняет форму для команды API, где указывает контакты клиента, общую суть проекта, желаемые сроки приобретения договора на kit;
* организация звонка с командой API для понимания сути проекта и обсуждения процесса в целом (обсуждение необходимости приобретения Kit или ее отсутствие);
* определение срока, на который будет приобретен kit, а также общего roadmap;
* подписание договора и выставление счета;
* организация ТГ канала, где решаются только быстрые вопросы;
* подключение к Pyrus для решения остальных вопросов;
* по окончании срока договора на kit, организация финального звонка и обсуждение результатов;
* отметка о завершении работы с клиентом.

## Что вы получаете 

### Помощь на начальном этапе 

* Знакомство с командой iikoAPI и обсуждение задач интеграции.
* Знакомство с архитектурой iiko, консультация по методам реализации поставленных задач.
* Разбор практических кейсов.

### Помощь на этапе разработки 

* Предоставление виртуальных серверов для организации тестовой среды.
* Консультирование по документации API.
* Разбор логов ошибок в рамках проекта интеграции с привлечением команд разработки.

### Помощь на этапе продуктизации 

* Ревью готового проекта, рекомендации по продукту и рекламным материалам.
* Подготовка к вебинару, ревью материалов, проверка канала связи.
* Размещение информации о решении на партнерском канале и организация вебинара для партнеров.
