В настоящем разделе описан процесс создания бота для отслеживания заказов доставки.

##  Шаги для выполнения

1. Выполнить [авторизацию](https://api-ru.iiko.services/docs#tag/Authorization) в iikoCloud API **/api/1/access\_token.** Срок жизни токена - 1 час. Токен нужно хранить у себя и по истечение его действия повторять запрос заново.
2. Выполнить [запрос организаций](https://api-ru.iiko.services/docs#tag/Organizations/paths/~1api~11~1organizations/post) **/api/1/organizations**. Запрос необходимо сделать лишь один раз, сохранив у себя идентификатор организаций. При необходимости (внесении изменений) нужно периодически обновлять.
3. Подписаться на [веб-хуки](/smart/project-api-documentation/webhooks). В теле веб-хука приходит состав заказа и номер заказа, по которому можно отслеживать его статус. Необходимо отсортировать нужные данные, к примеру, если интересуют только новые заказы - отслеживать нужно по статусу New.

iikoCloud API отсылает веб-хук к API telegram бота, а API ТГ бота в свою очередь передает информацию далее в сам telegram бот.
