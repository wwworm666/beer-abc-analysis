## Какие обращения мы принимаем 

### Новый продукт или интеграция 

Первое, что вам надо сделать: изучить статью по [реализации интеграционных проектов](/articles/api-documentations/~realizatsiya-integratsionnykh-proektov), заполнить Анкету и прислать ее нам на почту [api@iiko.ru](mailto:api@iiko.ru).

Постарайтесь как можно подробнее описать ваш продукт, который вы хотите создать на базе нашего API или интегрировать с ним. Мы постараемся вам подсказать, какие API вам нужно реализовать в том или ином случае для реализации требуемого функционала.

###  Проблема или ошибка 

Главное правило: одна заявка — одна проблема.

Не нужно заводить заявку «У нас возникло несколько проблем» и перечислять их. В 99% случаев проблемы никак не связаны друг с другом и требуют разного решения.

Перед созданием заявки:

1. Проанализируйте проблему.
2. Уточните модуль, в котором наблюдается проблема.
3. Посмотрите файлы логов проблемного модуля на предмет ошибок.

Если вы затрудняетесь, то опишите, что происходило до возникновения ошибки. Например:

* Обновились на новую версию: когда, какая версия была до этого.
* Меняли настройки томката или базы.
* Были известные проблемы на хостинге у других клиентов.
* Меняли какие-либо другие настройки.
* Начали использовать новый функционал.
* Подключили новое предприятие в iikoChain.

## Что писать в обращении

### Описание проблемы

Старайтесь описать проблему, насколько это возможно: что вы делаете, что получаете, что хотите получить.

Пример проблемы.

**Дата, время — они ДОЛЖНЫ быть в логах.**

**В описании укажите:**

Версию используемого ПО iiko: полная версия вида **Х.Х.ХХХХХ.Х**.

Если проблема возникла после обновления, укажите полную версию, с которой обновлялись.

При разборе проблем с API iikoCard для нас важны версии iikoRMS и POS-сервера. Например, в формате: iikoRMS xxxx POS xxxx.

Где можно посмотреть:

* в Windows-разделе "Программы"
* в логах POS-сервера при перезапуске сервера
* в начале ежедневных логов.

### Частота возникновения проблемы

Как часто сталкиваетесь с этой проблемой: один раз, редко, часто (как часто?).

### Время проблемы

Укажите даты и время, когда проблема появилась.

### Возможные связанные события

* Обновили версию: с какой версии на какую.
* Меняли настройки: какие.
* Начали использовать новый функционал.

### Попытки исправить проблему

Что вы уже делали по данной заявке: восстанавливали бекап базы, переименовывали папки, переустанавливали приложение и т.д.

###  Особенности клиента

Если пароли нестандартные – указать.

### Приложите логи компонента

Фронт / бэк / сервер / инсталлятор. После сохранения заявки убедитесь, что приложенные файлы есть в заявке.

#### Как понять, какие логи нужно приложить?

**Если вы используете наш Демо-Стенд и у вас возникли проблемы/вопросы по использованию** **API Server или Cloud API, то необходимо приложить следующую информацию:**

**Если обращение по API Server:**

* %programfiles%\iikoRMS\server\logs
* full.log
* error.log
* jms.log
* Запрос API , который вы делали и ответ на него (присылать в текстовом формате )
* CRMID или id организации
* если вопрос связан с данными из iikoOffice - приложить скриншот

**Если обращение по Cloud API:**

* error.log с iikoFront
* plugin-Resto.Front.Api.iikoTransport.log
* Запрос API , который вы делали и ответ на него в формате json(присылать в текстовом формате )
* CRMID или id организации
* При проблеме создания доставочного заказа / заказа в стол прикладывайте ответ на метод [https://api-ru.iiko.services/api/1/commands/status](https://api-ru.iiko.services/#tag/Operations/paths/~1api~11~1commands~1status/post)

Логи необходимо прикладывать в заархивированном виде. Предпочтительные форматы: zip, 7z, rar.

Не нужно прикладывать логи отдельными текстовыми файлами.

Ограничение на размер одного прикрепляемого файла – 4Мб.

**Если вы обращаетесь по готовой интеграции, и у вас возникают ошибки при работе API по действующему клиенту, то задачу необходимо отправлять к партнеру (техническую поддержку ), который обслуживает клиента.**

**Для анализа проблем с интеграционными продуктами** на базе iikoFront API или Payment API:

* Из папки %AppData%\iiko\CashServer\Logs\ все файлы вида \*.log за время проблемы или файлы логов , завернутые в архив с датой проблемы в имени файла.
* Все \*.xml файлы из папки %AppData%\iiko\CashServer\.
* Иногда папка CashServer фронта, снятая целиком утилитой Диагностика.

Info: %AppData%\ - папка Application data из профиля текущего пользователя, под учетной записью которого запускается программа iikoFront.

**Для анализа проблем с оборудованием** (Принтеры, Фискальные регистраторы, дисплеи покупателя, весы и другое торговое оборудование) укажите производителя и точную модель оборудования. Версию драйвера, тип подключения и предоставить логи фронта:

* Из папки %AppData%\iiko\CashServer\Logs\ все файлы вида \*.log за время проблемы или файлы логов , завернутые в архив с датой проблемы в имени файла.
* Обязательно лог агента ( с датой в имени, в архив не заворачивается) либо ДАТА-agent.log либо в новых версиях ДАТА-devices.log. Заявка без него бессмысленна.
* devices.xml и config.xml фронта.
* Лог драйвера оборудования, если он ведется.

**Если вы используете API iikoFront и он установлен в режиме агента**(в списке сервисов и служб присутствует запись iikoAgent) и служба запускается от имени локальной системы, то необходимо искать логи в зависимости от версии операционной системы( статья 206 БЗ):

* Windows 7/2008/10/11 x32: C:\Windows\System32\config\systemprofile\AppData\Roaming\iiko\CashServer
* Windows 7/2008/10/11 x64 также нужны логи из папки: C:\Windows\SysWOW64\config\systemprofile\AppData\Roaming\iiko\CashServer

## Удаленное подключение

| ![Warning](/resources/Storage/api-documentations/api-documentations/warning.png) | Удаленное подключение возможно только при использовании платного тарифа поддержки.<br><br>Техническая поддержка по TeamViewer версий выше 13 не может быть предоставлена в силу лицензионных ограничений TeamViewer. |
| --- | --- |

Если проблема требует нашего подключения к точке, предоставить удаленное подключение.

Подключение в порядке предпочтения возможно по TeamViewer, RDP, Ammyy Admin.

При предоставлении доступа указывайте постоянный пароль на TV. Если такового нет (клиент не предоставляет / еще не создан / и т.п.) – создайте. Техподдержка или разработчики не могут мгновенно подключиться к точке — есть большая вероятность, что временный пароль будет недействителен во время подключения.

Если кто-то уже подключен по RDP к предоставляемому подключению, Teamviever не сможет отобразить рабочий стол.

Для получения удаленной помощи запустите модуль для возможности удаленного подключения через TeamViewer QuickSupport (входит в поставку iikoRMS с версии 3.6 и выше. Находится в iikoRMS – TeamViewer, также кнопки вызова добавлены в приложения iikoFront и iikoRMS BackOffice).

Если у вас версия ниже и программы нет, [скачайте](http://download.teamviewer.com/download/version_13x/TeamViewerQS.exe) с официального сайта.

Если мы просим приложить какие-то дополнительные файлы, необходимо написать об этом в сообщениях и открыть заявку. Иначе нам не придет оповещение о том, что в заявке что-то добавилось и наша реакция на заявку замедлится.

Если заявка находится в статусе «Ожидает информации», не присылайте нам сообщения вида «Ок, сейчас сделаем» и подобные. Это только откроет заявку и потребует от нас отправки вам такого же лишенного смысла сообщения «Ждем», «Ок» и т.п.

Отправляйте нам сообщения только тогда, когда вы готовы предоставить запрошенную нами информацию.
