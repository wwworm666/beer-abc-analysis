API iiko позволяет интегрировать ресторанные процессы с внешними системами: сайтом, мобильным приложением, CRM, учетной системой или внутренними сервисами.
С помощью iiko API можно автоматизировать работу с меню, заказами, доставкой, клиентами и программами лояльности.

1. [Регистрация в кабинете разработчика](/articles/api-documentations/instruktsiya-po-rabote-s-portalom-razrabotchika-iiko)
    Сначала зарегистрируйтесь на портале разработчика iiko и создайте приложение.
После регистрации вы получите параметры, необходимые для авторизации:

    * appId (ID приложения)

    * clientSecret

![](/resources/Storage/api-documentations/polucheniyu-dostupa-k-api/polucheniyu-dostupa-k-api-2026-06-02-1.png)
 ![](/resources/Storage/api-documentations/polucheniyu-dostupa-k-api/polucheniyu-dostupa-k-api-2026-06-02-2.png)
2. Получить API ключ: [Подключение, получение API ключа](/articles/api-documentations/podklyuchenie-poluchenie-api-klyucha)
3. Так же необходимо ознакомиться со статьями [Сроки действия API ключа](/articles/api-documentations/sroki-deystviya-api-klucha) и [Ограничение и лимиты](/articles/api-documentations/ogranichenie-i-limity)
4. Авторизуйтесь по следующему методу [https://api-ru.iiko.services/api/v2/access_token](https://api-ru.iiko.services/docs#tag/Authorization/paths/~1api~1v2~1access_token/post). 
    Для получения токена используйте актуальный endpoint: https://api-ru.iiko.services/api/v2/access\_token

    В запросе передавайте appId и clientSecret, полученные при регистрации приложения.

    После успешной авторизации сервис вернет access token, который используется в последующих запросах к API.
5. После получения токена рекомендуется выполнить тестовый запрос (например, получение справочной информации), чтобы убедиться, что:

    * авторизация проходит успешно;

    * токен принимается сервером;

    * приложение корректно обрабатывает ответ API.

## FAQ по изменениям в системе авторизации iiko API

| ![Information](/resources/Storage/api-documentations/project-iikoweb/info.png) | Ответ на вопросы №2 и №7 обновлен 20.05.2026 |
| --- | --- |

1. Как быть поддержке? Если рейт-лимиты привязаны к приложениям в кабинете, то десятки сотрудников будут слать запросы под одним appId. Предусмотрены ли для поддержки повышенные лимиты?

Ответ: На данном этапе лимиты (Rate Limits) не меняются. Требуется только передавать ID приложения при авторизации. В случае изменения лимитов мы проинформируем вас дополнительно.

2. Как регистрироваться клиентам не из РФ? Какой адрес портала разработчика для СНГ (по аналогии с порталом iiko)?

Ответ: На данный момент регистрация является **обязательной для всех пользователей**. Портал разработчиков будет доработан в ближайшее время, чтобы вы могли ввести реквизиты, соответствующие законодательству вашей страны. Новость об обновлении портала разработчика мы обязательно опубликуем отдельно. Пожалуйста, ожидайте официального анонса.

3. Будет ли доступна авторизация для поддержки (L3 и партнеров) через старый метод? Возможно ли сделать временные ключи (на час/сутки)?

Ответ: Старый метод будет недоступен. Мы прорабатываем вопрос создания системных appId и clientSecret специально для нужд поддержки.

4. Правильно ли мы понимаем, что это требуется только для самостоятельных разработок клиентов? С октября старый эндпоинт для iikoTransport будет отключен?

Ответ: Изменения касаются как самостоятельных разработок клиентов, так и решений технологических партнеров. Старый эндпоинт действительно будет отключен, необходимо перейти на новый.

5. Затронут ли нововведения действующие интеграции типовых партнерских решений (бронирование, лояльность, CRM, QR-меню и т.д.)?

Ответ: Да, это относится ко всем партнерам и интеграциям, использующим iikoTransport. Типовым решениям необходимо перейти на новый метод авторизации.

6. Я — интегратор для нескольких разных фирм. Мне нужно регистрировать одно приложение как разработчику или создавать отдельное приложение для каждого клиента?

Ответ: Вам как разработчику достаточно один раз получить appId и clientSecret для своей интеграции (продукта). Далее вы получаете API-ключи у каждого конкретного клиента и используете их вместе со своим appId в запросах. Тиражировать интеграцию на разных клиентов можно с одним набором данных разработчика.

7. Пришло уведомление о необходимости регистрации, но я представляю юрлицо — резидента Республики Беларусь (РБ). В форме регистрации требуются ИНН и ОГРН, которых у нас нет. Как нам зарегистрироваться?

Ответ: На данный момент регистрация является **обязательной для всех пользователей**. Портал разработчиков будет доработан в ближайшее время, чтобы вы могли ввести реквизиты, соответствующие законодательству вашей страны. Новость об обновлении портала разработчика мы обязательно опубликуем отдельно. Пожалуйста, ожидайте официального анонса.

8. Правильно ли я понимаю: меняется только способ авторизации, а все остальные методы (эндпоинты) для работы с данными остаются прежними?

Ответ: Да, всё верно. Изменения затрагивают только метод получения токена доступа. Все остальные методы API остаются без изменений и продолжают работать в штатном режиме.

FAQ будет дополняться.
