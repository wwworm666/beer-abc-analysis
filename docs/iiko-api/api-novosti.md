В данной странице будут публиковаться важные изменения в работе API.

## Разработчикам из стран СНГ стала доступна регистрация на Портале 

На Портале разработчика открыта регистрация для разработчиков из стран СНГ.
Теперь специалисты и компании из региона могут самостоятельно создавать аккаунты, получать доступ к документации и инструментам платформы, а также разрабатывать и развивать собственные интеграции.
Подробнее в статье: [Инструкция по работе с порталом разработчика iiko](/articles/api-documentations/instruktsiya-po-rabote-s-portalom-razrabotchika-iiko)

## Изменения с 1 июня 2026 года

С 01.06.2026 действует новая схема авторизации. Для стабильной работы ваших интеграционных сервисов их нужно зарегистрировать на портале разработчика iiko. 
Старый метод авторизации прекращает работать:
 - с 01.06.2026 не принимаются вновь созданные API-ключи
 - с 29.08.2026 он будет полностью отключен
 Пожалуйста, проверьте все ваши интеграции в соответствии с инструкцией [https://ru.iiko.help/articles/#!api-documentations/polucheniyu-dostupa-k-api](/articles/api-documentations/polucheniyu-dostupa-k-api "https://ru.iiko.help/articles/#!api-documentations/polucheniyu-dostupa-k-api"). Это займет несколько минут, но сэкономит гораздо больше времени позже: если отложить – интеграция может перестать работать. Лучше обновиться сейчас.
 Переход на новый метод авторизации в API обеспечит не только стабильную работу ваших интеграций, но и доступ к новым возможностям и поддержке со стороны iiko.
## Изменения с 1 мая 2026 года

С 1 мая 2026 года перестаем поддерживать адреса в формате "legacy" в методе Cloud API deliveries/create

Необходимо перевести ваши интеграции на формат адреса "city". 
Новый формат адреса "city" работает эффективнее и предполагает создание заказа даже если в справочнике улиц на фронте отсутствует нужная улица. Геокодирование адресов в формате "city" проходит точнее.
**Что требуется:**- Уведомите интеграторов или разработчиков о необходимости изменения передачи формата адреса в интеграции с iiko.  
- Свяжитесь с вашим партнером, чтобы изменить настройку формата адреса в iiko.

**Подробная инструкция как работать с новым форматом адресов**

[https://ru.iiko.help/articles/api-documentations/vygruzka-gorodov-i-ulits/a/h3\_\_438059702](/articles/api-documentations/vygruzka-gorodov-i-ulits/a/h3__438059702)

Это поможет избежать возможных неполадок в будущем и сохранить все функции создания заказов. Пожалуйста к этому времени переведите свои интеграции на новый формат работы с адресом

## Изменения с 1 января 2026 года

С 01.01.2026 у клиентов с локальной версией iiko перестанут работать все облачные сервисы (например, iikoCard) и интеграции, подключенные через облачные API (например, перестанут поступать заказы на доставку из внешних источников). Рекомендуем вам перейти на облачную версию iiko.

Данные изменения не затронут API iikoFront и iikoServer.

## Изменения с 1 июля 2025 года

С 01.07.2025 начинает работу механизм блокировки не отвечающих обработчиков (эндпоинтов для получения веб хуков). Замечено большое количество ошибочных обработчиков, которые генерируют неоправданную нагрузку на транспорт, угрожая его стабильности. Такие обработчики будут заблокированы.

Правила корректной работы, а также критерии блокировки описаны в статье [Обработчики веб-хуков](/articles/project-api-documentation/~obrabotchiki-veb-hukov).
