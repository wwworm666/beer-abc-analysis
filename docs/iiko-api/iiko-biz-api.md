## Условия использования API iikoBiz и API iikoCard

С целью предотвращения спама запросами и защиты от ошибок в интеграциях в API iikoBiz и API RMS установлены искусственные лимиты для ограничения количества запросов от пользователей.

### Ограничение на количество запросов 
Лимиты применяются на конкретные апи-логины пользователей.
Cтандартный лимит равен 6 запросам в секунду, с допустимым запасом до 1500 запросов сверх этого лимита, к методу GetOrderInfo запас составляет до 6000 запросов.
Если отправлять больше 6 запросов в секунду, запас начнет расходоваться. Как только запас запросов будет исчерпан, в ответ на запросы будет выдаваться ошибка с кодом 429 и текстом: ***Too many requests, wait 500ms to continue***.
Запас запросов автоматически восстанавливается по 6 единиц в секунду. Таким образом, полное восстановление запаса в 1500 запросов займет 250 секунд (4.17 минут), если в это время не выполнять запросы.
Например, если запас был исчерпан, но теперь вы выполняете по 4 запроса в секунду, запас будет восстанавливаться по 2 единицы в секунду. Если продолжаете отправлять по 6 или больше запросов в секунду - запас восстанавливаться не будет.
