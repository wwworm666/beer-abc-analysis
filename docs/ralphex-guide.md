# Инструкция: Ralphex

Инструмент для автономного выполнения задач через Claude Code — читает план, выполняет задачи, делает код-ревью.

## Что это

Ralphex оркестрирует Claude Code: берёт markdown-файл с задачами, запускает каждую в отдельной Claude-сессии, коммитит результат, затем проводит многоэтапное код-ревью.

## Установка

Установлен в `C:\Users\1\go\bin\ralphex.exe` (v0.26.2). Если `ralphex` не виден из терминала, используй полный путь:

```powershell
& "$env:USERPROFILE\go\bin\ralphex.exe"
```

Для удобства добавь в `$PROFILE` (PowerShell):

```powershell
$env:PATH += ";$env:USERPROFILE\go\bin"
```

## Быстрый старт

### 1. Инициализация конфига

В корне проекта:

```powershell
& "$env:USERPROFILE\go\bin\ralphex.exe" /init
```

Создаст `.ralphex/` с настройками по умолчанию (прокомментируемыми).

### 2. Создание плана

**Интерактивно через AI:**
```powershell
& "$env:USERPROFILE\go\bin\ralphex.exe" /plan "описать что нужно сделать"
```

Ralphex изучит код, уточнит детали через меню и сформирует план.

**Вручную** — файл `docs/plans/название-плана.md`:

```markdown
# Plan: Добавить экспорт CSV

## Validation Commands
- `python manage.py test`

### Task 1: Создать view для экспорта
- [ ] Добавить endpoint `/api/export/csv`
- [ ] Покрыть тестами

### Task 2: Кнопка на UI
- [ ] Кнопка "Экспорт" в шапке таблицы
- [ ] Обработчик скачивания
```

Формат: `### Task N:` + чекбоксы `[ ]` (отмечаемые по мере выполнения).

### 3. Запуск

```powershell
& "$env:USERPROFILE\go\bin\ralphex.exe" docs/plans/название-плана.md
```

Без аргументов — покажет список существующих планов.

## Фазы выполнения

1. **Task Execution** — каждая задача выполняется в свежей Claude-сессии. После каждой — валидационные команды и коммит.
2. **First Review** — 5 агентов-ревьюеров запускаются параллельно:
   - **Quality** — баги, безопасность, race conditions
   - **Implementation** — задачи выполнены согласно плану
   - **Testing** — покрытие и качество тестов
   - **Simplification** — нет ли излишнего усложнения
   - **Documentation** — обновлена ли документация
3. **External Review** (опционально) — внешний ревьюер (другой AI или кастомный скрипт)
4. **Second Review** — 2 финальных агента (quality + implementation), только критические проблемы
5. **Finalize** (опционально) — ребейз/сквош коммитов

## Режимы запуска

| Команда | Описание |
|---------|----------|
| `ralphex plan.md` Полный цикл | |
| `ralphex /tasks-only plan.md` | Только выполнение задач, без ревью |
| `ralphex /review` | Только ревью текущего состояния |
| `ralphex /external-only` | Только внешний ревью |
| `ralphex /worktree plan.md` | В изолированном git worktree |
| `ralphex /serve /port 8080` | Веб-дашборд для мониторинга |

## Многоагентное ревью (дебаты)

У Ralphex есть встроенная **первая фаза ревью** — 5 агентов работают параллельно. Claude проверяет их замечания, фиксит подтверждённые, коммитит fixes.

Можно **настроить своих агентов** в `.ralphex/`.

### Кастомные агенты

По умолчанию агенты лежат в `pkg/config/defaults/agents` (в исходном коде). Для кастомизации:

1. Создай `.ralphex/agents/` в проекте
2. Определи агента через `{{agent:name}}` в промптах
3. Настрой свои промпты в `.ralphex/prompts/`

Можно запустить **внешний ревью** через любую команду — в т.ч. через другой `ralphex` с другим конфигом:

### Внешний ревью другим AI (через скрипт)

Файл `review.sh` (или `.bat` на Windows):

```bash
#!/bin/bash
# Запустить Codex / другой AI в режиме review
echo "Review changes..."
git diff
# Здесь команда обращения к другому AI для review
```

В конфиге укажи этот скрипт как внешний ревьюер. Это создаёт "дебаты" — Ralphex-агенты нашли проблемы, внешний AI их проверяет, Claude-основной решает.

## Флаги (Windows — через `/`)

| Флаг | Описание |
|------|----------|
| `/plan "текст"` | Интерактивное создание плана |
| `/init` | Инициализация `.ralphex/` конфига |
| `/tasks-only` | Только задачи |
| `/review` | Только ревью |
| `/external-only` | Только внешний ревью |
| `/worktree` | Изолированный worktree |
| `/skip-finalize` | Пропустить финализацию |
| `/max-iterations 20` | Лимит итераций |
| `/session-timeout 30m` | Таймаут сессии Claude |
| `/serve /port 8080` | Веб-дашборд |
| `/debug` | Отладочный вывод |
| `/base-ref main` | Базовая ветка для diff |

## Прерывание и продолжение

`Ctrl+\` (SIGQUIT) — пауза в середине выполнения. Можно отредактировать план, потом продолжить.

## Переменные окружения

Использует те же `ANTHROPIC_BASE_URL` и `ANTHROPIC_API_KEY` что и Claude CLI — твой ключ от работодателя работает автоматически.

## Веб-дашборд

```powershell
& "$env:USERPROFILE\go\bin\ralphex.exe" docs/plans/plan.md /serve /port 8080
```

Открой `http://127.0.0.1:8080` — видишь вывод с таймстампами, цветами и логами в реальном времени.

## Полезные команды

**Посмотреть список планов:**
```powershell
& "$env:USERPROFILE\go\bin\ralphex.exe"
```

**Резет конфига к дефолту:**
```powershell
& "$env:USERPROFILE\go\bin\ralphex.exe" /reset
```
