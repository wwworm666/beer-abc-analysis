# Beer Menu Card Creator

## Что это

Веб-интерфейс для создания пивных меню в формате A4-карточек (одна карточка = одно пиво). Фиксированный дизайн с точным повторением вёрстки при любом контенте (длина заголовка, описания, размер логотипа). Экспорт в PNG двумя способами: client-side (html2canvas) и server-side (Playwright/headless Chromium).

## Файлы

| Файл | Назначение |
|------|-----------|
| `routes/menu.py` | Blueprint: CRUD API, загрузка лого, рендер через Playwright |
| `templates/menu.html` | Страница конструктора: форма + live-превью + список позиций |
| `templates/menu_card.html` | Отдельный шаблон карточки (для standalone preview / Playwright) |
| `static/js/menu.js` | Встроен в menu.html (inline script) |
| `static/menu/logos/` | Загруженные логотипы |
| `data/menu_items.json` | Хранилище позиций (JSON-файл) |

## Как работает

### Структура карточки

Каждая карточка — A4 (1240x1754px) с фиксированными элементами:
- **Угловые акценты** — 60x60px рамки в каждом углу (opacity 0.3)
- **Разделители** — горизонтальные линии 80% ширины (opacity 0.1)
- **Логотип** — круг 200x200px, object-fit: cover
- **Заголовок** — PT Serif 110px, bold, uppercase
- **Тип пива** — PT Serif 52px, letter-spacing 0.3em
- **Происхождение** — PT Serif 44px, italic, 50% opacity
- **Описание** — PT Serif 72px, italic
- **Теги** — PT Serif 48px, bold, uppercase
- **Цены** — до 2 объёмов, разделённых вертикальной линией

### Правила заполнения карточки

1. **Шестая строка — вкусовые дескрипторы (теги):** всегда ровно три дескриптора, разделённых символом ` · ` (средняя точка с пробелами). Пример: `КАРАМЕЛЬ · ХЛЕБ · ТРАВЫ`
2. **Седьмая строка — цены:** по умолчанию всегда два объёма — `0,3 л` и `0,5 л`. Каждый с собственной ценой.

### Загрузка логотипа

Логотип можно загрузить тремя способами:
- **Выбор файла** — клик по зоне логотипа открывает файловый диалог
- **Drag & Drop** — перетащить изображение в зону логотипа
- **Вставка из буфера** — `Ctrl+V` в любом месте страницы загружает изображение из буфера обмена

### Flexbox-раскладка

`justify-content: space-between` на flex-контейнере гарантирует что контент распределяется по высоте карточки независимо от объёма текста. Длинные слова переносятся через `word-break: break-word`.

### Два режима рендеринга

| Режим | Метод | Требования |
|-------|-------|-----------|
| Client | html2canvas на DOM | Только браузер |
| Server | Playwright Chromium | `pip install playwright && playwright install chromium` |

Server-рендер даёт точную копию (1:1) без артефактов шрифтов. Client-рендер работает без зависимостей.

### API

| Endpoint | Method | Описание |
|----------|--------|----------|
| `/menu/` | GET | Страница конструктора |
| `/menu/api/items` | GET | Список всех позиций |
| `/menu/api/items` | POST | Создать позицию |
| `/menu/api/items/<id>` | PUT | Обновить позицию |
| `/menu/api/items/<id>` | DELETE | Удалить позицию |
| `/menu/api/logo` | POST | Загрузить логотип (base64 или multipart) |
| `/menu/api/render` | POST | Рендер карточки в PNG (Playwright) |
