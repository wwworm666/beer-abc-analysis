## 6️⃣ ОСНОВНЫЕ РИСКИ

### 🔴 Критические риски (могут увеличить сроки на 30-50%):

1. **Нестабильность iiko API** (+40-60ч)
   - Изменения в формате ответов
   - Недокументированные лимиты
   - Таймауты при больших запросах
   - *Митигация:* Кэширование, retry логика, детальное логирование

2. **Сложность мэппинга кегов** (+30-50ч)
   - Кириллица ↔ латиница
   - Ошибки в названиях от поставщиков
   - Постоянное добавление новых сортов пива
   - *Митигация:* Fuzzy matching, ручной CSV интерфейс, документация

3. **Изменение требований в процессе** (+50-80ч)
   - "А давайте ещё добавим..."
   - Изменение методологии ABC/XYZ
   - Новые виды отчётов
   - *Митигация:* Детальное ТЗ, согласование на этапе проектирования

### 🟡 Средние риски (могут увеличить сроки на 15-30%):

4. **Проблемы с производительностью** (+20-30ч)
   - Большой объём данных (4 бара × 1 год)
   - Медленные OLAP запросы
   - *Митигация:* Индексы, кэширование, пагинация

5. **Cross-browser совместимость** (+15-25ч)
   - Разное поведение в Safari/Chrome/Firefox
   - Мобильные браузеры
   - *Митигация:* Раннее тестирование на разных браузерах

6. **Недостаток документации iiko API** (+20-30ч)
   - Reverse engineering
   - Trial-and-error подход
   - *Митигация:* Поддержка от iiko, форумы, тестовая среда

### 🟢 Минорные риски (могут увеличить сроки на 5-15%):

7. **Технический долг** (+10-20ч)
   - Спешка на начальном этапе
   - Необходимость рефакторинга
   - *Митигация:* Code review, тестирование

8. **Проблемы с Render** (+10-15ч)
   - Ограничения бесплатного тира
   - Необходимость миграции на paid план
   - *Митигация:* Заложить бюджет на хостинг
