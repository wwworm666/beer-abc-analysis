## 📋 Что мы знаем точно

### 1. API Endpoints

| Endpoint | Метод | Описание |
|----------|-------|----------|
| `/api/v3/true-api/auth/key` | GET | Получение пары `uuid` + `data` для подписи |
| `/api/v3/true-api/auth/simpleSignIn` | POST | Обмен подписанных данных на токен |

### 2. Формат ответа `/auth/key`
```json
{
  "uuid": "a63ff582-b723-4da7-958b-453da27a6c62",
  "data": "GNUFBAZBMPIUUMLXNMIOGSHTGFXZMT"
}
```
- `uuid` — идентификатор сессии аутентификации
- `data` — строка для подписи (30 символов, заглавные латинские буквы)

### 3. Формат запроса `/auth/simpleSignIn`

**Противоречие в документации:**

| Версия API | Формат запроса |
|------------|----------------|
| v2 (docs.cr11pt.ru.html) | `{"uuid": "string", "data": "string", "inn": "string"}` |
| v3 (docs.crp333t.ru.html) | `{"data": "string", "unitedToken": true}` |

**Цитата из документации v3:**
> "Отправка в ГИС МТ данных в том же виде, в котором данные были получены (пара `uuid - data`), только теперь `data` — это подписанная УКЭП строка."

### 4. Требования к подписи

Из документации:
- `data` в запросе должна быть **подписанной УКЭП строкой**
- Подпись должна быть в формате **Base64 ОДНОЙ СТРОКОЙ** (без `\r\n`)
- Используется ГОСТ Р 34.10-2012 (256 бит)

## 🧪 Протестированные команды подписи

### Команда 1: `-sfsign -sign` (присоединённая подпись)
```cmd
csptest.exe -sfsign -sign -my "2297e52c..." -in data.txt -out sig.txt -base64
```
**Результат:** ✅ Подпись создаётся (1660 символов)
**API ответ:** ❌ 403 "Подпись невалидна. Код ошибки: 4"

### Команда 2: `-sfsign -sign -detached` (откреплённая подпись)
```cmd
csptest.exe -sfsign -sign -detached -my "2297e52c..." -in data.txt -out sig.txt -base64
```
**Результат:** ✅ Подпись создаётся (1616 символов)
**API ответ:** ❌ 403 "Подпись невалидна. Код ошибки: 4"

### Команда 3: `-sfsign -sign -cades -detached` (CAdES-BES)
```cmd
csptest.exe -sfsign -sign -cades -detached -my "2297e52c..." -in data.txt -out sig.txt -base64
```
**Результат:** ❌ Ошибка 160 "invalid option -- 'cades'"
**Проблема:** Флаг `-cades` не распознан версией CSP

### Команда 4: `-sign` (простая подпись без -sfsign)
```cmd
csptest.exe -sign -my "2297e52c..." -in data.txt -out sig.txt -base64
```
**Результат:** ✅ Подпись создаётся
**API ответ:** ❌ 403 "Проверка подписи не пройдена"

### Команда 5: `-hash` + подпись хэша
```cmd
csptest.exe -hash -in data.txt -out hash.txt
csptest.exe -sfsign -sign -my "2297e52c..." -hash hash.txt -out sig.txt -base64
```
**Результат:** ⚠️ Хэш создаётся, подпись хэша — ошибка доступа
**API ответ:** Не тестировалось

## 🔍 Ключевые наблюдения

### 1. Локальная проверка подписи НЕ ПРОХОДИТ
```cmd
csptest.exe -sfsign -verify -my "2297e52c..." -in data.txt -signature sig.txt -base64
```
**Ошибка:** `0x8009310b: Встречено неверное значение тега ASN1`

Это означает что подпись создаётся **технически невалидной** — возможно из-за:
- Неправильного формата данных для подписи
- Проблемы с кодировкой (UTF-8 vs Windows-1251)
- Отсутствия необходимых атрибутов в PKCS#7

### 2. Разные коды ошибок API
- **Код 4:** "Подпись невалидна" — формат подписи неверный
- **Код 2:** "Проверка подписи не пройдена" — подпись валидна но не подходит для данных

### 3. Проблема с доступом к csptest.exe
При запуске из Python (subprocess) — **"Отказано в доступе"**
При запуске из cmd напрямую — **работает нормально**

## 🎯 Гипотезы для проверки

### Гипотеза 1: Кодировка данных
Возможно данные нужно подписывать в кодировке **Windows-1251** вместо UTF-8:
```python
with open(data_file, 'wb') as f:
    f.write(data_to_sign.encode('cp1251'))  # вместо utf-8
```

### Гипотеза 2: Данные с newline
Возможно в конце строки нужен `\n` или `\r\n`:
```python
f.write((data_to_sign + '\n').encode('utf-8'))
```

### Гипотеза 3: Требуется полный PKCS#7 с атрибутами
Возможно ЧЗ требует **CAdES-BES с атрибутами** (время подписи, сертификат):
```cmd
csptest.exe -sfsign -sign -cades -my ... -attr -out sig.txt
```

### Гипотеза 4: Другой базовый URL
Возможно для v2 API нужен другой URL:
```
https://markirovka.crpt.ru/api/v2/true-api  # вместо v3
```

### Гипотеза 5: Требуется промежуточный сертификат
Возможно цепочка сертификатов неполная и ЧЗ не может верифицировать.

## 🛠️ Следующие шаги для проверки

### 1. Проверить кодировку данных
```cmd
# Сохранить в UTF-8
echo|set /p=DATA > data_utf8.txt
# Сохранить в CP1251 (через PowerShell)
powershell -command "[System.IO.File]::WriteAllText('data_cp1251.txt', 'DATA', [System.Text.Encoding]::Default)"
# Подписать оба файла и сравнить
```

### 2. Проверить с newline
```cmd
echo DATA > data_with_newline.txt
echo|set /p=DATA > data_without_newline.txt
```

### 3. Использовать certmgr для подписи
```cmd
certmgr.exe -sign -cont "2508151514-781421365746" -in data.txt -out sig.txt -base64
```

### 4. Проверить цепочку сертификатов
```cmd
certmgr.exe -list -storem ca
certmgr.exe -list -storem root
```

### 5. Попробовать КриптоПро .NET SDK
Если csptest.exe не работает корректно, попробовать COM или .NET:
```csharp
CPSign cadesSign = new CPSign();
cadesSign.SignCades(data, CadesType.CadesXLong, ...);
```
