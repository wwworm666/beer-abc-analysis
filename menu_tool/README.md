# Beer Menu Card Tool — локальный

Standalone Flask-приложение для создания и печати пивных карточек
формата A4. Запускается **только локально** на ПК бара — не деплоится
на Render (Chromium-рендер ест ~150–200 МБ, не влезает в 512 МБ Starter).

## Что это умеет

- Библиотека позиций с поиском (`/menu/library`).
- Редактор одной карточки с live-превью (`/menu/?id=N` или пустой для новой).
- Глобальный переключатель **2 объёма (0,3 / 0,5)** ↔ **3 объёма (0,25 / 0,4 / 0,5)**.
- Экспорт **PDF (вектор)** — для печати. Текст и тонкие линии не
  растрируются, печатается на родном DPI принтера (600–1200 DPI) без
  потерь. `page.pdf(format=A4, margin=0, print_background=True,
  prefer_css_page_size=True)` — настройки печати пользователя не
  применяются, выход идентичен спецификации.
- Экспорт **PNG (растр)** — для соцсетей/мессенджеров. 1588×2246 px
  ≈ 192 DPI. Для печати **не годится** — драйвер принтера тянет
  растр на родное разрешение (300+ DPI), текст блюрится.

Дизайн зафиксирован спецификацией Claude Design «Beer Menu Spec — 2 and
3 volumes»; правила вёрстки и формулы — в исходниках шаблонов.

## Установка (один раз)

```
cd menu_tool
py -m pip install -r requirements.txt
py -m playwright install chromium
```

Chromium-бинарник кешируется в `~/.cache/ms-playwright/` (Windows:
`%USERPROFILE%\AppData\Local\ms-playwright\`), ~150 МБ.

## Запуск

```
cd menu_tool
py app.py
```

Открыть в браузере: <http://127.0.0.1:5050/menu/library>

Порт **5050** (не 5000) — чтобы не пересекаться с основным аналитическим
Flask-приложением проекта.

## Как лежат данные

```
menu_tool/
├── app.py                    # Flask на 127.0.0.1:5050
├── menu_routes.py            # Blueprint: страницы, CRUD, PNG-рендер
├── templates/
│   ├── menu.html             # Редактор: форма + переключатель A/B + превью
│   ├── menu_library.html     # Таблица со всеми карточками + поиск
│   └── menu_card.html        # Шаблон одной A4-карточки (для превью и PNG)
├── data/
│   ├── menu_items.json       # Все карточки. Источник истины.
│   └── menu_settings.json    # { "variant": "v2" | "v3" }
├── requirements.txt          # flask, playwright
└── README.md                 # этот файл
```

Данные **версионируются вместе с репо** — если редактируешь карточки
локально и хочешь поделиться правками, делай `git commit -m "menu: …"` и
`git push`. Если несколько человек правят менюшку в разных копиях — могут
быть конфликты в `menu_items.json`; разруливай вручную.

## Модель данных (одна позиция)

```json
{
  "id":      1,
  "n":       3,
  "name":    "Коммерцинрат",
  "latin":   "Riegele Commerzienrat",
  "brewery": "Riegele",
  "country": "Германия",
  "style":   "Немецкий лагер",
  "abv":     "5,2",
  "tags":    ["солодовый", "медовый", "благородный"],
  "ratings": { "gor": 2, "plot": 3, "cvet": 2 },
  "p025": 470, "p03": null, "p04": 670, "p05": 770
}
```

Десятичный разделитель — **запятая** (ru-RU). `null` в ценах рендерится
как `—`. Знак валюты не выводится — это часть дизайна.

## Варианты разлива

| Вариант | Объёмы | Когда |
|---|---|---|
| A (`v2`) | 0,3 / 0,5 | Если бар не разливает 0,25 |
| B (`v3`) | 0,25 / 0,4 / 0,5 | По умолчанию |

Переключатель глобальный (хранится в `menu_settings.json`) — одна и та же
схема для всего распечатанного буклета.

## API endpoints

| Endpoint | Method | Что |
|---|---|---|
| `/menu/` | GET | Редактор (опционально `?id=N`) |
| `/menu/library` | GET | Библиотека всех карточек |
| `/menu/card?id=N&variant=v2\|v3` | GET | Standalone превью одной карточки |
| `/menu/api/items` | GET / POST | Список / создание |
| `/menu/api/items/<id>` | PUT / DELETE | Обновление / удаление |
| `/menu/api/settings` | GET / PUT | Глобальный вариант |
| `/menu/api/render` | POST | Рендер карточки в **PNG** через Playwright (растр для соцсетей) |
| `/menu/api/render-pdf` | POST | Рендер карточки в **PDF** через Playwright (вектор для печати) |

## Печать

**Для печати используй PDF** (PNG для печати не годится — растрируется,
шакалит при масштабировании драйвером принтера на родное разрешение).

1. Открой `/menu/?id=N`.
2. Нажми **PDF (A4)** — скачается файл `menu-XX-Name-vN.pdf` ~30–50 КБ.
3. Открой PDF в любом просмотрщике → Печать → A4, **«Размер 100%»**
   (НЕ «По размеру страницы»), без полей. Adobe Acrobat и Foxit делают
   это автоматически по A4-метаданным; в Edge/Chrome убедись что
   «Масштаб = 100%».
4. Поскольку CSS уже указывает `@page { size: A4; margin: 0 }` и PDF
   создан с теми же параметрами — макет не должен съезжать.

**Альтернатива через браузер** (не рекомендую, но если PDF почему-то не
открывается): открыть `/menu/card?id=N` в Chrome → Ctrl+P → «Сохранить
как PDF». Учти что пользовательские поля могут сдвинуть макет (отключи
«Колонтитулы» и «Поля → нет»).
